# MCP Inspector Quickstart

## Purpose
Use this as the shortest read-first path in MCP inspector clients.

## Portable Notes
- This quickstart is client-agnostic and does not depend on any machine-local paths.
- Read the live resources first; do not guess the public MCP surface from the filesystem.
- The repo root `server.json` is the matching public-safe metadata summary for this server surface.
- `MCP_REGISTRY_READINESS.md` is the matching public-safe readiness note for future distribution work.
- `MCP_DISTRIBUTION_QUICKSTART.md` is the matching shortest public-safe first hop for installer-facing or consumer-facing distribution work.
- `MCP_REGISTRATION_GUIDANCE.md` is the matching clone-agnostic registration guide for client setup.

## Quick Path
1. Read `server_inventory` to confirm the live surface.
2. Read `read_order` to follow the repo's recommended inspection sequence.
3. Read `consumer_router` to choose the first safe consumer hop.
4. Read `path_conventions` to keep paths repo-relative and clone-agnostic.
5. Read `prompt_selection_guide` to choose the smallest safe prompt before drafting work.
6. Read `task_map` to jump from a common task to the smallest helpful resource.
7. Read `surface_map` to jump from a surface to the matching doc or test.
8. Read `tool_selection_guide` to pick the narrowest safe tool.
9. Read `client_usage_guide` for the recommended flow.
10. Use a draft prompt before a direct write tool.
11. Validate before apply.
12. If the task is still unclear, fall back to `server_inventory` and `tool_selection_guide` again instead of guessing.
13. Use `wechat://installer-readiness` when you need the installer-facing public distribution checklist.
14. Use `wechat://distribution-quickstart` when you need the shortest public-safe starting point for installer or consumer distribution work.
15. Use `wechat://registration-guidance` when you need the clone-agnostic registration guide for client setup.
